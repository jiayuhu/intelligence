这是一份第一轮真实 `AI行业情报` 的总入口页，按这个顺序打开，其余材料都作为子文档使用。

## 各文档作用
- `first-run-checklist.md`：总执行顺序
- `execution-cards.md`：按主题逐卡执行
- `topic-matrix.md`：主题、来源、关键词、要点矩阵

- `search-queries.md`：可执行搜索语句（含扩展搜索语句）
- `source-sites.md`：具体站点和页面清单（含扩展站点）
- `sources.md`：来源优先级与扫描顺序（含扩展来源 v1.0）
- `extended-sources.md`：**扩展来源清单**（新增 36 个来源 + 社交媒体）
- `extended-source-checklist.md`：**扩展来源接入检查清单**
- `social-media-sources.md`：**社交媒体来源清单**（X / LinkedIn 账号）
- `first-run-source-log.md`：人工检索记录模板
- `report-outline.md`：成稿章节骨架
- `report-sample.md`：成稿样例
- `email-summary-sample.md`：邮件摘要样例
- `first-run-delivery-checklist.md`：最终交付检查

## 第一轮的核心原则
- 只抓 48 小时内的信息
- 优先官方来源和高可信来源
- `AI Agent`、`AI Coding`、头部 AI 企业、AI 领袖人物优先
- **扩展来源**：新兴企业（xAI, Cursor）、Agent 基础设施、技术领袖扩展
- **社交媒体**：X 官方账号和核心领袖作为补充（P1），LinkedIn 企业动态作为参考（P2）
- 不确定信息统一标注为"待确认"
- 成稿结构严格跟随 `report-outline.md`

## 扩展来源核心原则（v1.0 新增）
- **扩展来源优先级**：P0 扩展 = 原有 P0，P1 扩展 = 原有 P1
- **冲突处理**：扩展来源与原有来源冲突时，优先采信原有来源
- **扫描顺序**：先完成原有 P0-P4，再扫描扩展来源
- **质量标注**：扩展来源统一标注 `source_tier: P1/P2`

## 社交媒体核心原则（新增）
- **作为补充**：社交媒体信息作为官方渠道的补充，不单独作为主要来源
- **交叉验证**：X/LinkedIn 信息需与官方来源或高可信媒体交叉验证
- **领袖观点**：领袖人物观点需标注"个人观点，不代表公司立场"
- **平台标注**：统一标注 `source_platform: x/linkedin`
- **谨慎处理**：未确认账号信息标记为"待确认"

## 交付目标
第一轮结束时，应至少产出：
- 抓取 JSON
- Markdown 成稿
- 发送 JSON
- HTML 报告
- PDF 文件
- 邮件 HTML 预览
- `.eml` 邮件包
