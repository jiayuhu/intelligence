# AI行业情报第一份成稿提纲

## 目标
这份提纲用于第一份 `AI行业情报` 的成稿结构，方便抓取结果直接映射到正文。
它同时作为 `collect.md` 和 `draft.md` 的对齐基准。

## 封面信息
- 标题：`AI行业情报`
- 副标题：`48小时滚动监测`
- 日期：报告生成日期
- 时间窗口：`48` 小时

## 一、本期要点

### 一句话总览
用一句话（不超过 50 字）概括本期最核心的变化，包含：主体 + 动作 + 意义。

### 关键数据
3-4 条数据概括本期情报概况：
- 本期监测情报总数、已确认数、待确认数
- 覆盖的主要主题
- 高可信度来源数量

### 要点列表
输出 3-5 条精炼要点，每条格式：

**【主题标签】结论标题**（15 字以内）
一句话说明核心变化（聚焦事实，不写背景、依据、影响）。

要求：
- 使用主题标签：【AI Agent】【AI Coding】【公司动态】【领袖观点】【模型/基建】【政策监管】
- 优先覆盖 AI Agent、AI Coding、AI 公司、领袖人物
- 只陈述事实，不展开分析（分析放到后面主题章节）
- 不重复：这里提到的事件，在主题章节详细展开

## 二、AI Agent
- 1 段总述（50-80 字）
- 2-5 条动态

动态格式（4 行/条）：
```
**[MM-DD] 事件标题** | [重要度] [状态] [来源等级]
- 详情：[具体描述事件，约 300 字，包含功能细节、适用场景、技术实现等]
- 影响：[一句话说明业务/竞争含义]
- 来源：[来源名称](链接)
```

示例：
```
**[04-05] GitHub 推出 Copilot 组织级防火墙** | 高 已确认 tier1
- 详情：GitHub 为 Copilot Cloud Agent 新增 organization 级别的防火墙设置，这是企业级 AI Agent 治理能力的重要升级。新功能允许企业管理员在组织层面统一配置网络边界，包括 IP 允许列表、出口规则和网络访问控制。管理员可以精细控制哪些 cloud agent 可以访问内部资源、外部 API 以及特定域名。该功能与同期发布的 Runner Controls 和签名提交支持形成完整的企业安全体系，适用于需要满足合规要求的大型企业。目前该功能已向 GitHub Enterprise Cloud 客户开放，预计未来两周内全面推送。
- 影响：AI Agent 治理从可选项变为必选项，企业选型时将更关注治理能力而非仅功能
- 来源：[GitHub Blog](https://...)
```

## 八、社区热点追踪
- 1 段总述（30-50 字）
- 3-5 条热点

社区热点格式（3 行/条）：
```
**[平台] 热点话题** | [热度] [情绪]
- 概况：[一句话描述讨论内容和核心分歧]
- 信号：[对行业的早期信号或警示]
```

热度：高/中/低（根据讨论量、互动数）
情绪：正面/负面/争议/观望

示例：
```
**[Reddit/HN] OpenAI 降价引发开发者讨论** | 高 争议
- 概况：开发者社区对 OpenAI 降价 25% 反应两极，一方欢呼成本下降，另一方担忧初创公司生存空间被压缩
- 信号：价格战可能比预期来得更快，应用层公司需加速寻找差异化定位
```

## 十、信息缺口
- 仅列出当前还缺失、但值得继续追踪的问题
- 说明需要继续观察的来源或人物
- 不写泛泛的"继续关注"

## 十一、邮件发送摘要
结构化邮件摘要，适合快速扫读：
- **本期概览**：数据卡片（情报数、确认数、覆盖主题）
- **核心看点**：3-5 条，每条约 30-50 字，带关键影响
- **一句话洞察**：50-80 字概括本期最大趋势
- **建议行动**：3-4 条，分优先级（高/中），含负责角色
- **附件信息**：PDF 文件名、在线版本链接

## 第一份成稿的写作重点
- AI Agent 和 AI Coding 必须持续占据核心位置
- 主要 AI 公司和领袖人物必须优先于泛行业内容，同时关注创新型 AI 公司
- 只写 48 小时内可确认的信息
- 任何不确定内容都标注为"待确认"
