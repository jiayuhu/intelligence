## 任务目标
基于抓取结果 JSON 生成正式情报成稿，并确保所有结论都能追溯到输入中的事实条目。
成稿必须严格以 `report-outline.md` 为骨架，章节顺序和命名不得自行扩展或改写。
正文中只允许出现 `report-outline.md` 中已有的章节标题和其附属小节，不新增替代标题。

## 输入要求
你将收到一份符合 `collect-output-schema.json` 的 JSON 数据，顶层结构包含：
- `report_title`
- `report_date`
- `time_window_hours`
- `generated_at`
- `groups`

每个 `group` 包含：
- `category`
- `summary`
- `items`

每个 `item` 包含：
- `title`
- `published_at`
- `source_name`
- `source_url`
- `subject`
- `classification`
- `event_summary`
- `why_it_matters`
- `confidence`
- `related_focus`
- `within_48h`
- `status`
- `notes`
- `source_tier`

输入变量名为 `{{fetchResultJson}}`，如果某个字段缺失或为空，应在成稿中以“待确认”处理，不得自行补写。

## 写作要求
- 全文使用中文
- 语言专业、简洁、克制
- 面向决策者，不写空泛背景
- 明确区分“事实 / 推断 / 建议”
- 每条结论都尽量说明依据
- 对 AI Agent 和 AI Coding 的变化要持续深挖，不要只停留在表面新闻
- 对 AI 公司和领袖人物要强调战略含义
- 先写结论，再写证据，再写影响
- 只使用输入 JSON 中的事实，不引入新信息
- 相同事件的多个来源应合并，不重复展开

**写作要求**：

## 二、AI Agent
先写 1 段总述（50-80 字），概括本期该领域的核心变化。

再列 2-5 条动态，使用**紧凑格式**（4-5 行/条）：

```
**[MM-DD] 事件标题** | [重要度] [状态] [来源等级]
- 主体：[subject]
- 详情：[event_summary，100 字以内，具体描述发生了什么]
- 影响：[why_it_matters，一句话说明业务/竞争含义]
- 来源：[source_name](source_url)
```

**字段说明**：
- `[MM-DD]`：抓取结果中的 `published_at`，只保留月-日
- `事件标题`：用 `title` 或基于 `event_summary` 提炼（20 字以内）
- `[重要度]`：高/中/低
- `[状态]`：已确认/待确认
- `[来源等级]`：tier1/tier2/tier3 或 P0/P1/P2/P3
- `详情`：具体描述事件，基于 `event_summary`，保留关键细节（功能/数值/范围等）
- `影响`：聚焦业务/竞争含义，不写背景

**示例**：
```
**[04-05] GitHub 推出 Copilot 组织级防火墙** | 高 已确认 tier1
- 主体：GitHub / Microsoft
- 详情：GitHub 为 Copilot Cloud Agent 新增 organization 级别的防火墙设置，企业管理员可统一配置网络边界、IP 允许列表和出口规则。
- 影响：AI Agent 治理从可选项变为必选项，企业选型时将更关注治理能力
- 来源：[GitHub Changelog](https://github.blog/...)
```
