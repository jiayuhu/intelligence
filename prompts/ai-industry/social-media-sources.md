## 三、社交媒体采集指南

### 3.1 采集优先级

| 优先级 | 平台 | 账号类型 | 使用场景 |
|--------|------|----------|----------|
| P1 | X | 官方企业账号 | 产品发布、API 变更 |
| P1 | X | 核心领袖人物 | 战略表态、观点变化 |
| P2 | X | 技术领袖/研究员 | 技术趋势、研究动向 |
| P2 | LinkedIn | 官方企业页面 | 企业动态、招聘信号 |
| P2 | LinkedIn | 高层领袖 | 战略观点 |
| P3 | X | 媒体/VC 账号 | 市场风向 |
| P3 | LinkedIn | 群组/讨论 | 行业情绪 |

### 3.2 信息验证原则

| 来源类型 | 可信度 | 处理方式 |
|----------|--------|----------|
| 官方认证账号确认的消息 | 高 | 直接采用 |
| 领袖人物个人账号观点 | 中 | 标注为个人观点 |
| 未确认账号的爆料 | 低 | 标记待确认，需交叉验证 |
| 评论区的讨论 | 低 | 仅作线索 |

### 3.3 关键信号识别

**X 上的关键信号：**
- 🔥 预告推文（"明天有大事"）
- 📊 数据分享（用户增长、性能指标）
- 💼 招聘信息（暗示新产品方向）
- 🔗 链接分享（研究论文、博客）
- 💬 互动讨论（与其他领袖的互动）

**LinkedIn 上的关键信号：**
- 🏢 企业重大公告
- 👥 高管变动
- 📈 融资/增长数据
- 📝 长文观点分享
- 🎯 招聘信息（技术栈暗示）

### 3.4 技术实现建议

**X (Twitter) 采集：**
- 方式 1: X API v2 (付费，有速率限制)
- 方式 2: Nitter 实例 (免费，不稳定)
- 方式 3: 人工监测关键账号 (可靠但人力成本高)
- 方式 4: RSS 桥接 (如 rss.app)

**LinkedIn 采集：**
- 方式 1: LinkedIn API (需申请，限制较多)
- 方式 2: 公开页面抓取 (需处理反爬)
- 方式 3: 人工监测关键页面
- 方式 4: 邮件通知订阅

## 四、与其他来源的协同

### 4.1 冲突处理
| 场景 | 处理方式 |
|------|----------|
| X 预告早于官方博客 | 标注为"预告"，待官方确认 |
| LinkedIn 招聘 vs 官方新闻 | 招聘信号仅作线索，不单独成稿 |
| 领袖观点 vs 公司官方立场 | 领袖观点需标注个人立场 |
| X 讨论 vs 媒体报道 | 以媒体报道为准，X 作补充 |

### 4.2 信息增强
| 场景 | 协同方式 |
|------|----------|
| 官方博客发布后 | 查看 X 上的用户反馈 |
| 产品发布时 | 查看 LinkedIn 上的企业动态 |
| 融资消息时 | 交叉验证 X 和 LinkedIn |
| 人事变动时 | LinkedIn 个人页面更新通常最早 |
