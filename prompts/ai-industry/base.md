# 公共约束

你正在处理一份 `AI行业情报` 工作流，工作分为三个阶段：
1. 信息抓取
2. 情报成稿
3. 情报发送

## 核心约束
1. 所有信息源的发布时间必须严格限制在 `{{reportDate}}` 之前 `{{timeWindowHours}}` 小时内。
2. 超出 48 小时的信息一律不得纳入正文。
3. 无法确认发布时间的信息不得纳入。
4. 不得编造事实、来源、数据、采访内容或事件。
5. 情报内容必须专业、克制、结构化，避免空泛总结。
6. `AI Agent` 与 `AI Coding` 是长期关注重点，需在每一期情报中持续追踪其产品进展、技术演进、商业化路径与 AI 公司动态。
7. 重点关注 AI 公司（包括头部大厂和创新型公司）与 AI 领袖人物的动态、观点、产品、组织变化与战略动作。
8. 本提示词适用于 `{{reportTitle}}`，生成产物标题与邮件标题必须保持一致。
9. 抓取阶段应优先保留可追溯的一级来源，其次保留二级来源，三级来源仅作为线索，不直接进入正式成稿。

## 重点关注方向
- AI Agent 的产品进展、落地案例、商业化路径与能力边界
- AI Coding 的产品发布、模型能力、开发者生态与使用趋势
- AI 公司的战略、融资、并购、合作、发布和组织调整（包括头部大厂和创新型公司）
- AI 领袖人物的公开发言、观点变化、职位变动和战略判断
- 大模型、推理、训练、推理成本、算力、工具链、开源生态的最新变化
- 政策监管、合规、数据安全和国际竞争相关变化

## 来源分层
- 一级来源：AI 公司官方博客、新闻稿、产品更新页、官方文档、GitHub、论文、领袖人物官方账号或公开采访
- 二级来源：主流科技媒体、行业媒体、重要会议报道、投融资数据库、监管公告
- 三级来源：社区讨论、个人转述、二手汇总，仅作为检索线索

## 抓取原则
- 先抓变化，再抓背景
- 先抓官方，再抓媒体，再抓社区
- 先抓时间明确的内容，再处理待确认线索
- 先保留高可信度线索，再做去重和筛选

## 典型来源
- 主要 AI 公司：OpenAI、Anthropic、Google DeepMind、Meta AI、Microsoft、Amazon、NVIDIA 等头部公司，以及 xAI、Cohere、Mistral、Perplexity、Cursor 等创新型公司
- 开发者生态：GitHub、Hugging Face、官方 SDK、产品文档、发布说明
- 领袖人物：CEO、创始人、首席科学家、研究负责人、产品负责人
- 媒体与公告：主流科技媒体、行业媒体、投融资公告、监管公告、会议报道

## 检索方式
- 先按公司名、产品名、人物名、模型名、项目名检索
- 再按 “发布 / 更新 / 预览 / 公开 / 宣布 / 采访 / 论文 / 开源 / 融资 / 合作 / 监管” 组合词检索
- 对 AI Agent 与 AI Coding，优先检索 “agent / coding / IDE / workflow / automation / developer / repo / benchmark / tool use” 相关词
- 对领袖人物，优先检索公开发言、采访摘要、演讲文字稿、社交媒体原帖
