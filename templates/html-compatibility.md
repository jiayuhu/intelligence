# HTML 兼容规范

本仓库的 HTML 输出同时面向 Web 页面、Outlook 邮件、阿里邮箱和网易邮箱。默认约束如下：

## 结构要求
- 使用 `table` 作为主布局，不依赖 `div + flex/grid` 作为主结构
- 所有关键样式使用内联样式
- 保持内容结构简单、层级稳定
- 页面宽度优先使用固定容器加自适应外层包装
- 报告页和邮件页应复用同一母版结构，避免不同模板出现布局分叉

## 样式要求
- 允许：字体、字号、颜色、间距、边框、背景色、行高、对齐、内边距
- 尽量避免：`position`、`float`、`flex`、`grid`、复杂动画、外链字体、脚本
- 优先使用保守的 CSS，避免依赖不同客户端兼容性差异较大的属性
- 文本区应保留换行和自动折行，避免内容溢出
- 关键容器建议显式使用 `table-layout:fixed`

## 邮件客户端注意事项
- Outlook 兼容优先
- 表格建议显式声明 `cellspacing="0"`、`cellpadding="0"`、`border="0"`
- 建议补充 `mso-table-lspace:0pt` 和 `mso-table-rspace:0pt`
- 不使用外部资源、脚本和表单

## 内容一致性
- 同一份 HTML 在 Web、Outlook、阿里邮箱和网易邮箱中应显示相同的标题、元信息和正文内容
- 不通过客户端差异化逻辑改变内容顺序
- 若必须裁剪样式，优先裁剪视觉效果，不裁剪信息内容
